根类别 (Root Categories)

发表于 2016-11-21 分类于 Magento 2 用户使用手册

原文地址

类别结构像是一棵倒置的树，根在最上面。根类别基本上是主菜单的容器。根类别有名字但没有 URL key，而且他不出现在店铺顶端的导航中。其他类别被嵌套组织在根类别下。由于根类别是类别的最高层，所以店铺一次有且仅有一个根类别处于激活状态。不过，你可以创建另外的根类别备用，或者针对不同的店铺和视图。

第一步：创建一个根类别

  1. 后台侧边栏，点击 Products，在 Inventory 下，选择 Categories
  2. 设置 Store View 为你的更类别可以被使用的视图。
  3. 点击 Add Root Category
  4. 完成以下基本信息：
    a. 设置 Enable Category 为 Yes
    b. 让顶部导航包含该根类别，设置 Include in Menu 为 Yes
    c. 输入根类别的 Category Name
  5. 点击 Save 然后继续。

第二步：添加子类别到新的根类别

  1. 在左侧的类别树中，选择你上面刚刚创建的根类别
  2. 点击 Add Subcategory
  3. 完成所有新类别都要填写的信息。
  4. 重复以上步骤，创建需要的子类别。

第三步：将新的根类别应用到店铺上

  1. 后台侧边栏，点击 Stores，在 Settings 下，选择 All Stores
  2. 在 Store 下，点击 Main Website Store 链接
  3. 设置 Root Category 为上面创建的新的根类别。
  4. 完成后，点击 Save Store

    a. 在 Design Rules 下，点击 Add New Agent Rule
    b. 在 Search String 栏中，输入特定设备的 browser ID
    c. search string 可以是普通的表达式或者是 Perl Compatible Regular Expression ，要了解更多，请参考 User Agent，下面的表达式代表 Firefox:

    d. 在 Theme Name 栏中，选择你要应用的主题

    e. 重复上述步骤输入其他规则，search strings 按照他们被输入的顺序进行匹配。

  5. 根据需要进行其他设置

    • HTML Head
    • Header
    • Footer
    • Pagination
    • Product Image Watermarks

新产品列表 (New Products List)

发表于 2016-11-15 分类于 Magento 2 用户使用手册

原文地址

新产品列表是说明动态内容的好例子，他是从产品库中取出的实时的数据。默认情况下，新产品列表包含的是最新添加的八个产品。但是，也可以通过配置让他只包含某个时间段内的产品。

第一步：设置每个产品的日期范围

要让产品按照时间范围出现在新产品列表中，您必须给产品填入 “New From” 日期。

  1. 后台侧边栏，点击 Products，在 Inventory 下，选择 Catalog
  2. 选择你要更改的产品进行编辑：
    a. 在 Set Product as New From Date 字段中，点击日历小空间，然后选择开始时间
    b. 在 Set Product as New To Date 字段中，点击日历小空间，然后选择结束时间
  3. 完成后，点击 Save
  4. 系统提示你清理缓存，照做。
    现在新产品列表中就会出现在时间范围内的产品了。

第二步：创建小部件

决定新产品列表位置的代码由 Widget 工具生成。

  1. 后台侧边栏，点击 Content，在 Elements 下，选择 Widgets

  2. 右上角点击 Add New Widget

  3. 在设置部分，做如下事情：
    a. 设置 Type 为 Catalog New Products List
    b. 选择 Design Theme 为当前主题

  4. 点击 Continue

  5. 在 Storefront Properties 部分，完成下列字段：

    Widget Title

    输入描述标题，这个标题只在后台可见

    Assign to Store Views

    选择 widget 可用的 store view

    Sort Order

    （可选）输入数字决定次序（0 = 第一个，以此类推）

第三步：选择位置

  1. 在 Layout Updates 部分，点击 Add Layout Update，然后做如下事情：
    a. 设置 Display On 为 Specified Page
    b. 设置 Page 为 CMS Home Page
    c. 设置 Block Reference 为 Main Content Area
    d. 设置 Template 为下列之一：

    • New Products List Template
    • New Products Grid Template

  2. 点击 Save and Contine Edit

第四步：配置列表

  1. 在左边的面板中选择 Widget Options

  2. 设置 Display Products 为下值之一：

    All Products

    列表中显示最近添加的产品

    New Products

    只显示标记为新的的产品，当前时间在产品是否是新的的时间范围内的才显示，所以如果这样设置，可能列表是空的

  3. 完成剩下的信息：
    a. 要给多页的情况提供导航，设置 Display Page Control 为 Yes
    b. 设置 Number of Products to Display 为新产品的数量，默认是 10 个
    c. 在 Cache Lifetime (Seconds) 中输入你希望多久刷讯一次列表，默认是 86400 秒，也就是 24 小时

  4. 完成后，点击 Save

  5. 按照提示刷新缓存。

第五步：预览

  1. 后台侧边栏，点击 Content，在 Elements 下，选择 Pages
  2. 找到新产品列表显示的页面，进行预览。

page types

发表于 2016-11-15 分类于 Magento 2 高级

Page types are semantic abstractions of those controller actions which are used for page rendering. They help to qualify pages by context type, like Product View page, Shopping Cart page, and so on, providing business users the ability to customize page design. In particular, page types are used in the widget creation to define the pages where a widget can be displayed.

Page types are defined for the frontend area only, that is for the store front pages.

创建小部件 (Creating a Widget)

发表于 2016-11-15 分类于 Magento 2 用户使用手册

原文地址

每种类型的小部件的创建过程都是差不多的。

第一步：选择类型

  1. 后台侧边栏，选择 Content，在 Elements 下选择 Widgets
  2. 右上角点击 Add Widget，然后做如下事情：
    a. 在 Settings 部分，设置 Type ，然后点击 Contine
    b. 检查下 Design Theme 设置为当前的主题。
    c. 点击 Contine
  3. 在 Storefront Properties 中，做如下事情：
    a. 在 Widget Title 中输入描述性的标题。
    b. 设置 Assign to Store Views 为 All Store Views 或者是你想要他可用的 view。多选的话，请按住 Ctrl 键。
    c. 在 Sort Order 中输入数字。这个数字决定着如果相同位置有其他元素，他的位置。最上面的位置对应着 0。

第二步：指定放在哪里

  1. 在 Layout Updates 部分，点击 Add Layout Update
  2. 设置 Display On to 为你想要该 block 呈现的 category, product 或 page。
  3. 在 Block Reference 列表中，就是下图的 container，选择页面布局的位置
  4. 如果小部件是一个链接，那么设置 Template 为下列值中的任意一个：
    Block Template: 他将作为独立的块显示
    Inline Template: 可以作为行内元素显示，比如在一段文本中

第三步：完成选项

每种类型的小部件的选项差别很小，过程本质上是一样的。下面的例子展示了如何显示特定目录的产品列表，并且带有分页控件。

    1. 左侧的小面板中，选择 Widget Options
  2. 点击 Select Block，在列表中选择你想要的 block
  3. 分页控件，设置 Display Page Control 为 Yes，然后做下面的事情：
    a. 输入 Number of Products per Page（每页的产品数目）
    b. 输入 Number of Products to Display （总共展示的产品数目）
    c. Set Condition to the category of products to be featured. The process is the same as setting a condition for a price rule.(有点奇怪没有找到对应的功能)
  4. 完成后，点击 Save
  5. 系统会提示你清理缓存，照做。

第四步：检查一下

去前台检查一下是否正确，如果要修改，重新编辑即可。

小部件类型 (Widgets Types)

发表于 2016-11-15 分类于 Magento 2 用户使用手册

原文地址

TYPE

DESCRIPTION

CMS Page Link

Displays a link to a selected page. Allows you to specify custom text and title. When the link is complete, it can be used in content pages and blocks.（展示一个链接到特定页面的链接）

CMS Static Block

Displays a block of content at a specific location on a page.（在页面的特定位置展示内容块）

Catalog Category Link

Displays either an inline or block-style link to a selected catalog category. When the link is complete, it can be used in content pages and blocks.（展示到某个产品分类的链接）

Catalog New Products List

Displays a block of products which have been designated as new, for the duration of time specified in the product record.（展示被指定为新产品的产品列表）

Catalog Product Link

Displays either an inline or block-style link to a selected catalog product. When the link is complete, it can be used in content pages and blocks.（展示某产品的链接）

Catalog Products List

Displays a list of products from the catalog.（展示产品列表）

Orders and Returns

Gives customers the ability to submit their own requests for returned merchandise authorization.（让客户可以提交请求退货）

Recently Compared Products

Displays the block of recently compared products. You can specify the number of products included, and format them as a list or product grid.（展示客户最近比较过的产品）

Recently Viewed Products

Displays the block of recently viewed products. You can specify the number of products included, and format them as a list or product grid.（展示客户最近浏览的商品）
