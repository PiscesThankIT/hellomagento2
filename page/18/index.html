捆绑商品(Bundle Product)

发表于 2017-01-04 分类于 Magento 2 用户使用手册

原文地址

捆绑商品让客户从各种各样的选项中“创造他们自己的”商品。捆绑产品可以是礼品、电脑、或者其他可以自定义的商品。捆绑商品中的每个项可以是以下商品类型：

  • 简单商品
  • 虚拟商品

当客户点击 Customize and Add to Cart 按钮后，才会出现选型。因为商品包含在捆绑数组中，所以 SKU 、价格、重量 可以被设置为静态的或固定的

Minimum Advertised Price (MAP) is not available for Bundle products with dynamic pricing.

以下将指引你创建一个基本设置的捆绑商品。必填项以红色星号(*)标记出来。填完必填项并保存后，你可以继续填写剩下的字段。

第一步：选择产品类型

  1. 后台侧边栏，点击 Products ，在 Inventory 下选择 Catalog
  2. 右上角 Add Product 菜单，选择 Bundle Product

第二步：选择属性集

属性集是产品的模板，要选择属性集，可以在搜索框里输入属性集的名字。

第三步：完成必要的设置

  1. 输入产品的名称 Product Name

  2. 使用基于产品名称的默认 SKU 或者重新输入一个 SKU
    决定分配给每个捆绑项的 SKU 的类型，可以做如下事情：

    • A Dynamic SKU can be assigned automatically to each bundle item by adding a suffix to the default SKU. By default, Dynamic SKU is set to “Yes.”
    • If you prefer to assign a unique SKU for each bundle item, set Dynamic SKU to the “No” position.

  3. 决定捆绑商品的价格，可以做如下设置：

    • A Dynamic Price changes to reflect the options chosen by the customer. By default, Dynamic Price is set to “Yes,” and the Price field is left blank.
    • To charge a fixed price for the bundle, set Dynamic Price to the “No.” position. Then, enter the Price that you want to charge for the bundle.

  4. 因为产品还没有准备好发布，所以把 Enable Product 切换到 No 的位置

  5. 点击 Save
    产品被保存后，左上角会出现 Store View chooser（用来选择作用于哪个店铺）

  6. 选择作用于那个 store view

第四步：完成基本设置

  1. 设置 Tax Class 为以下值之一：
    • None
    • Taxable Goods
  2. 输入库存数量 Quantity，注意默认情况下 Stock Status 设置为 In Stock
  3. 设置捆绑商品的重量，做如下事情：
    • A Dynamic Weight changes to reflect the options chosen by the customer. By default, Dynamic Weight is set to “Yes,” and the Weight field is left blank.
    • To assign a fixed weight to the bundle, set Dynamic Weight to the “No.” position. Then, enter the Weight of the bundle.
  4. Visibility 默认设置是 Catalog, Search
  5. 把产品分配给 Categories，点击选项框，这一步也可以创建 category ，都是显而易见的，所以不详细翻译了。
  6. 如果要标记改商品为新品，勾选 New 复选框
  7. 选择 Country of Manufacture

    可能还有很多其他属性。属性集的选项是不一样的，你可以稍后再填写他们。

第五步：添加捆绑商品

  1. 下拉到捆绑商品选项部分，然后，设置 Ship Bundle Items 为以下值之一

    • Separately
    • Together

  2. 点击 Add Option，做如下操作：

    a. 输入 Option Title 他将作为字段的标签名
    b. 设置 Input Type 为下列值之一：

    c. 如要设置字段必选，则勾选 Required 复选框
    d. 点击 Add Products to Option，然后选择你要包含的产品
    e. 点击 Add Selected Products

    f. 等列表出现后，选择其中一个作为默认项
    g. 输入 默认数量 Default Quantity (客户选择该项时的默认数量)
    h. To allow customers to change the quantity of any bundle item, mark the User Defined checkbox of any item that you do not want changed.
    By default, the Default Quantity that is included in a bundle item cannot be changed by the customer. However, the customer can enter the quantity of the item that is to be included in the bundle.
    For example, if the Default Quantity of the Sprite Status Ball is set to 2, and the customer orders 4 of that bundle option, the total number of total balls purchased is 8.

  3. 重复上述过程，给每个要添加到捆绑商品中的项做好设置

  4. 如要移除，点击 图标

第六步：填写商品信息

往下拉，然后根据需要填写以下部分：

  • Content
  • Images and Videos
  • Related Products, Up-Sells, and Cross-Sells
  • Search Engine Optimization
  • Customizable Options
  • Products in Websites
  • Design
  • Gift Options

第七步：发布商品

  1. 如果你已经准备好要发布产品了，那么设置 Enable Product 为 Yes
  2. 下面你可以点击右上角的 Save & Close 或者 Save 然后再 Admin 菜单中选择 Customer View 检查下产品发布信息是否 OK

Input Controls

Control

Description

Drop-down

Displays a drop-down list of options with the product name and price. Only one item can be selected.

Radio Buttons

Displays a radio button for each option, followed by the product name and price. Only one item can be selected.

Checkbox

Displays a checkbox for each option, followed by the product name and price. Multiple items can be selected.

Multiple Select

Displays a list of options with the product name and price. To select multiple items, hold down the Ctrl (or Option) key, and click each item.

Field Descriptions

Field

Description

SKU

Determines if each item is assigned a variable, dynamic SKU, or if a fixed SKU is used for the bundle. Options include: Fixed / Dynamic.

Weight

Specifies the weight is calculated based on the items selected, or is a fixed weight for the entire bundle. Options include: Fixed / Dynamic.

Price View

Determines if the product price is shown as a range, from the least expensive to the most expensive (Price Range), or with the least expensive shown (As Low As). Options include: Price Range / As Low As.

Ship Bundle Items

Specifies if individual items can be shipped separately.

要点

  • 客户可以构建他们自己的捆绑商品
  • 捆绑商品项可以是没有自定义选项的简单商品或虚拟商品
  • price view 可以被设置为价格区间或者是 As Low as (最低)
  • SKU 和 重量可以是固定的或动态的
  • 数量可以是预定义的或客户自己定义的值
  • 捆绑商品可以一起配送或分开配送

组合商品是一些独立的简单商品的组合，你可以提供简单商品的变体，或者按照季节、主题组合单品形成一套不同的商品。每个单品是可以单独购买的，或者作为组合商品的一部分购买。在购物车中，每个商品是单独列出的。

第四步：完成基本设置

  1. 设置 Tax Class 为以下值之一：
    • None

  3. 输入产品的重量
  4. Visibility 默认设置是 Catalog, Search
  5. 把产品分配给 Categories，点击选项框，这一步也可以创建 category ，都是显而易见的，所以不详细翻译了。
  6. 如果要标记改商品为新品，勾选 New 复选框
  7. 选择 Country of Manufacture

    可能还有很多其他属性。属性集的选项是不一样的，你可以稍后再填写他们。

第五步：将商品添加到组合中

  1. 下拉找到 Grouped Products 部分，然后点击 Add Products to Group
  2. 如有必要，可以使用筛选找出你需要包含的商品

  1. 后台侧边栏，点击 Stores，在 Settings 下，选择 Configuration
  2. 左侧边栏 Sales 下，选择 Checkout，展开 Shopping Cart 部分
  3. 设置 Grouped Product Image 为 Product Thumbnail Itself
  4. 完成后，点击 Save Config

要点

  • 组合商品本质上是简单商品的组合
  • 作为组合商品的一部分的简单商品和虚拟商品不能拥有自定义选项(custom options)
  • 组合商品中的每项被购买后都会单独存在于购物车中，而不是作为组合商品的一部分
  • 购物车中的缩略图，可以被配置为使用组合商品的图片，或是组合商品中单项的图片

可配置商品添加配置项(Adding Configurations)

发表于 2017-01-03 分类于 Magento 2 用户使用手册

原文地址

下面这个例子中添加了三个颜色三个尺码的配置项。总共会创建9个简单商品，每个商品都有唯一的 SKU ，页面顶部的进度条会显示你现在在哪里，并引导你完成所有步骤。

步骤一：选择属性

  1. 上一部分 可配置商品(Configurable Product) 点击保存后，下拉到 Configurations 部分，然后点击 Create Configurations
  2. 勾选你要包含的属性。比如，这里我们选择 color 和 size
  3. 列表显示了该属性集中所有可被应用在可配置商品上的属性
  4. 如果你需要添加新的属性，点击 Create New Attribute 完成attribute 的属性，然后点击 Save Attribute，然后勾选复选框选择属性。
  5. 右上角，点击 Next

步骤二：输入属性值

  1. 为每个属性勾选应用到该商品上的值
  2. 重新排列属性的顺序，可以拖动 图标到新的位置。他们的顺序决定着产品详情页中下拉列表的位置。
  3. 在进度条中，点击 Next

步骤三：配置图片、价格和数量

这一步决定着每个配置项的图片、价格和数量。可选项是一样的，你只能选一个。可以是 apply the same setting to all SKUs, apply a unique setting to each SKU, or skip the settings for now

  1. 选择应用的配置选项

    • 配置图片
      方法一：对所有 SKUs 应用同一组图片(Apply a Single Set of Images to All SKUs)

      1. 选择 Apply a Single Set of Images to All SKUs
      2. 浏览选择图片或者拖动图片到下图的框中

        方法二：对每个 SKU 应用单独的图片
        因为我们已经为父商品上传了图片，我们将使用这个选项来为每个颜色上传图片。假设某个客户购买了某个颜色的T恤，那么购物车中显示的就是这里的图片。
      3. 选择 Apply unique images by attribute to each SKU
      4. 选择图像要说明的属性，比如：color
      5. 对每个属性值，要么浏览选择图片，要么拖拽到框中
        If you drag the an image to a value box, it appears in the sections for the other values, as well.如果你想要删除图片，点击 图标

    • 配置价格
      方法一：对所有 SKU 应用相同的价格 (Apply the Same Price to All SKUs)

      1. 选择 Apply single price to all SKUs
      2. 输入 价格

        方法二：对每个 SKU 应用不同的价格 (Apply a Different Price for Each SKU)
      3. 如果商品的各种变体价格不一样，选择 Apply unique prices by attribute to each SKU
      4. 选择价格差异基于的属性
      5. 设置每个属性值的价格。在这个例子中，XL 尺码的更贵一点

    • 配置数量
      方法一：对所有的 SKU 应用相同的数量 (Apply the Same Quantity to All SKUs)

      1. 如果所有 SKU 的库存数量都是一样的，选择 Apply the Same Quantity to All SKUs
      2. 输入 数量

        方法二：不同的属性应用不同的数量 (Apply Different Quantity by Attribute)
      3. 如果每个属性的数量不同，选择 Apply Different Quantity by Attribute
      4. 设置每个的值
  2. 完成后，点击右上角的 Next

步骤四：生成产品配置

等一会儿，等产品列表显示出来以后，做如下事情：

  • 如果你对生成的配置满意，点击 Next
  • 要做修改，点击 Back

步骤五：添加产品图片

  1. 下拉到 Images and Videos 部分，然后展开
  2. 点击 相机 图标，选择该商品的主图
    了解更多，请查阅 Images and Video

步骤六：完成产品信息

下拉完成以下部分的信息

  • Content
  • Images and Videos
  • Related Products, Up-Sells, and Cross-Sells
  • Search Engine Optimization
  • Customizable Options
  • Products in Websites
  • Design
  • Gift Options

步骤七：发布商品

  1. 如果该商品可以发布了，则设置 Enable Product 为 Yes
  2. 你可以点击 Save & Close 或者以客户的视角浏览商品的信息

步骤八：配置购物车缩略图(可选)

如果你商品的每个变体都有不同的图片，你可以配置购物车的缩略图使用正确的图片

  1. 后台侧边栏，点击 Stores，在 Settings 下，选择 Configuration
  2. 左侧边栏 Sales 下，选择 Checkout，展开 Shopping Cart 部分
  3. 设置 Configurable Product Image 为 Product Thumbnail Itself
  4. 完成后，点击 Save Config

要点

  • 可配置产品允许客户从下拉列表中选择选项。每个选项实际上都是一个单独的简单商品
  • 每个下拉列表的值都是基于 dropdown 类型的属性。drop-down 类型的属性必须包含在属性集中，该属性集则是当前可配置产品的模板
  • 购物车中产品的缩略图可以配置为产品的图片，或者是可配置产品的变体产品的图片

简单商品(Simple Product)

发表于 2017-01-03 分类于 Magento 2 用户使用手册

原文地址

发挥产品类型的力量的关键之一在于学会什么时候使用简单商品类型。简单商品可以被单独出售，或者作为组合商品、可配置商品及捆绑商品的组成部分出售。拥有自定义选项的简单商品，有时候也被称为符合商品 (composite product)

以下步骤将带领你，使用默认的商品模板和基本的设置，创建一个简单商品。学会基本的设置后，你就可以根据需要完成更高级的设置了。

第一步：选择产品类型

  1. 后台侧边栏，点击 Products ，在 Inventory 下选择 Catalog
  2. 右上角 Add Product 菜单，选择 Simple Product

第二步：选择属性集

属性集是产品的模板，要选择属性集，可以在搜索框里输入属性集的名字。

第三步：完成必要的设置

  1. 输入产品的名称 Product Name
  2. 使用基于产品名称的默认 SKU 或者重新输入一个 SKU

  6. 选择作用于那个 store view

  1. 设置 Tax Class 为以下值之一：
    • None
    • Taxable Goods

  3. 输入产品的重量
  4. Visibility 默认设置是 Catalog, Search

商品类型 (Product Types)

发表于 2017-01-03 分类于 Magento 2 用户使用手册

原文地址

简单商品

简单商品是具有单独的 SKU 的实体。简单商品有各种定价和输入控件，这样就可以在出售时有一些变化。简单商品可以和组合商品、捆绑商品、可配置商品联系起来使用。

可配置商品

可配置商品看起来有一系列的选项。每个选项代表着单独的具有唯一 SKU 的简单商品。这样可以追踪每个变体的库存（比如具体到某件衣服某个颜色某个尺码的库存）

组合商品

组合商品是组合在一起进行出售的商品，有时候这样是为了促销。可以单独购买，也可以购买几种组合。

虚拟商品

虚拟商品是无形的，通常用于出售如下商品：服务、会员资格、担保、订阅的产品等。虚拟商品可以被应用在组合商品和捆绑商品中。

捆绑商品

捆绑商品让客户从各种各样的选项中“创造他们自己的”商品。捆绑产品可以是礼品、电脑、或者其他可以自定义的商品。捆绑商品中的每个选项都是单独的商品。

可下载商品

包含一个或多个可疑下载的电子文件的产品就是可下载的商品。这些文件可以放在自己的服务器上，或者是提供下载的 URL

Excel 编辑 csv 中文乱码

Magento 中很多地方用到 csv 格式的文件，在 Magento 2 中文化问题（原创） 中提到，对待 csv 文件，你可以使用任何编辑器进行编辑，比如 notepad、sublime text 之类的，或者专门的编辑器 Ron’s Editor。但是 Ron’s Editor 是收费的，破解版也不敢用，而一般的编辑器对付修改不太多的情况还是够用的，但是列一多的话，没有格子，弄起来很不方便。所以看起来还是用 Excel 比较好。

Excel 处理有中文字符的文件会出现乱码，本篇将介绍怎么处理这种情况。

出现乱码的原因是 Excel 以 ANSI 格式打开，不会做编码识别。而我们的 csv 文件是 UTF-8 编码的。

步骤

  1. 打开 Excel ，新建一个空白的工作簿，
  2. 数据 -> 自文本，选择 csv 文件
  3. 此处选择 utf-8 编码，下一步
  4. 勾选“逗号”，下一步，完成

这时候我们就可以编辑他了，最好另存为 Excel 格式的文件，保存中间状态，最后需要 csv 文件格式的时候再另存为 -> CSV (逗号分隔符)(*.csv)

建议使用的 Magento 配置

Magneto 为不同的使用情形提供了相当多的部署和配置选项。要充分发挥 Magento 本身的性能和超强的扩展性优势，需要正确设置好 Magento 。此外，关于环境配置的建议，不在本篇范围之内。

重要说明：文中涉及到的测试都是基于建议和支持的配置。配置的偏差可能导致不同的测试结果，不能准确反映出产品设计和构建的自然情况

核心配置

Magento 2.0 为使用 PHP7 和 Varnish 而优化，虽然也可以在 PHP5.6 和没有 Varnish 的情况下运行，不过不建议这样用。建议同时使用 PHP7 和 Varnish 来支持站点的业务处理，获得最佳的性能。

静态资源设置

静态资源的设置决定着 CSS,JavaScript,HTML 和图像等资源，如何准备好供站点使用。

  • Stores -> Configuration -> Advanced -> Developer:
    • Grid Settings: Asynchronous indexing: Enable
      • CSS Settings: Minify CSS Files: Yes
    • JS Settings: Minify JS Files: Yes
    • Merge JavaScript Files: No
    • Template Settings: Minify HTML: Yes
    • JavaScript Settings: Enable JavaScript Bundling: Yes

页面缓存

页面缓存设置决定着使用页面缓存的哪个文件版本，为了获得最佳性能，强烈建议开启 Varnish 的缓存。注意，这需要部署和配置 Varnish

  • Stores -> Configuration -> Advanced -> System -> Full page cache
    • Caching Application->Varnish Caching

Email 设置

Email 设置控制着邮件何时发送，它和邮件何时生成相关。

  • Stores->Configuration->Sales->Sales Emails:
    • General settings: Asynchronous sending: Enable

索引设置

  • System->Index Management:
    • All indexers should be in “Update on schedule” mode

应用缓存

  • Configure the platform to use Redis in env.php

分层架构

Magento 2.0 平台是分层的架构，公有四层，分别为性能和可扩展性进行了优化。他们是：the client, the page cache, the application, and the database tiers

此种标准的 web 应用结构，允许架构的不同组件独立地扩展，以应对不同的业务需求和不同的负载情况。

接下来的章节将描述 Magento 2.0 对每层做的性能和扩展性的改进

性能和扩展性的改进

Magento 2.0 改进了应用如何与浏览器进行交互，以及减少了服务器必须传输给浏览器进行渲染的数据量。这些改进可以显著地提高页面响应时间和加载时间，给建立更快的站点提供一个良好的起点。

客户端的改进

页面加载时间很重要，Magneto 通过减少页面渲染必须的数据量来降低加载时间。主要有以下方式：

  • 使用了压缩 （即删掉不必要的字符）。使用了 jShrink 和 cssmin，以及针对 HTML 的内部开发的压缩引擎。图片使用 PHP 的 gd2 库进行压缩。
  • 最大限度利用浏览器缓存。初次访问会缓存所有 Js 和 css，这样可以减少下个页面的请求。
  • 在结账和购物车操作中使用异步操作。

页面缓存

通过 Varnish page caching 来提供更快的服务器响应。
