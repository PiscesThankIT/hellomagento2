多个地址(Multiple Addresses)

发表于 2016-10-15 分类于 Magento 2 用户使用手册

原文地址

多个发货地址选项允许客户在结账时将一个订单发给多个地址，以及一个订单最多可以被发给多少个地址。

配置发货到多个地址

  1. 后台侧边栏，点击店铺(Stores) > 设置(Settings) > 配置(Configuration)
  2. 在左侧面板选择销售(Sales)> 多送货地址设置(Multishipping Settings)
  3. 展开 选项(Options) 做如下操作:
    a. 设置 允许发货到多个地址(Allow Shipping to Multiple Addresses )为 “Yes”
    b.输入允许发货到多个地址的最大数量(Maximum Qty Allowed for Shipping to Multiple Addresses)
  4. 完成后，点击 Save Config

配置建立从您的店铺或仓库发货的起点。起点也被用来确定销售商品的税率。

设置起点

  1. 后台侧边栏，点击店铺(Stores) > 设置(Settings) > 配置(Configuration)
  2. 在左侧面板选择销售(Sales)> 发货设置(Shipping Settings)
  3. 展开 起点(Origin) 完成如下内容：
    • Country
    • Region/State
    • ZIP / Postal Code
    • City
    • Street Address (and line 2, if needed)
  4. 完成后，单击 Save Config

体积重量(Dimensional Weight)

发表于 2016-10-15 分类于 Magento 2 用户使用手册

原文地址

体积重量指的是按照重量和体积计算运输费用，也是很常见的计算方法。特别是包裹比较轻但是比较占空间。

主流的物流公司现在都会采用按体积重量计算运费的。不过，体积重量的计算方法，物流公司之间不统一。因公司而异。

  • DHL
  • FedEx
  • UPS
  • USPS

我们建议您熟悉每家物流公司体积重量的算法。如果贵公司发货挺多的，那么算法的一个小小差别一年就可能差几千美元。Magento 本身的配置不包含对体积重量计算方式的支持。

表费率(Table Rates)

发表于 2016-10-14 分类于 Magento 2 用户使用手册

原文地址

费率表方式指的是根据一些条件，参考表的记录来计算运输费用，这些条件包括：

  • Weight v. Destination
  • Price v. Destination
  • # of Items v. Destination

举例来说，假设你的仓库在洛杉矶，那么发货到圣地亚哥就比到佛蒙特州要便宜。您可以使用费率表的方式给离您近的客户更多好处。

用来计算费率的数据是以电子表格的形式准备好后，导入网站的，计算出的结果显示在购物车的运费估算部分。

    c. 输入方法名(Method Name)，它会作为一个标签显示在购物车的计算费率旁边。

固定运费(Flat Rate Shipping)

固定运费是预先定义的固定的收费方式，可以按件或者按次计算。固定运费是一个简单计费方案，特别适合一些采用固定运费的物流公司。

设置固定运费

  1. 后台侧边栏，点击店铺(Stores) > 设置(Settings) > 配置(Configuration)

  2. 在左侧面板选择销售(Sales)> 送货方式(Shipping Methods)

  3. 展开 固定运费(Flat Rate) 做如下操作:
    a. 设置 启用(Enabled)为“Yes”

    b. 输入标题(Title)。
    c. 输入方法名(Method Name)，他会出现在购物车计算费率边上。默认的名字叫“Fixed”。如果您收取手续费，您可以把方法名改为“Plus Handling” 或者其他合适的名字。
    d. 类型(Type)中有如下选项，描述了固定运费的使用方式：

    name

    description

    None

    固定运费会被列出来，但费率为0，相当于包邮

    Per Order 每笔订单

    按订单计算

    Per Item

    按件计算，不管是不是同一种商品，每一件都计算

    e. 在价格(Price)中输入要收取的价格
    f. 如果收取额外手续费的话，在计算手续费(Calculate Handling Fee)中选择：

    然后在手续费(Handling Fee)中输入相应的值。比如，如果费用按百分比算，输入0.06表示百分之六。如果按固定的算法，输入数值。
    g. 在出错提示信息(Displayed Error Message)中输入固定运费方式出错时的提示信息。
    h. 设置哪些国家可用(Ship to Applicable Countries)

包邮

发表于 2016-10-14 分类于 Magento 2 用户使用手册

原文地址

免费送货是最有效的促销手段之一。可以在购买总额达到一定值后包邮，或者设置购物车价格规则，满足规则即可包邮。如果同一笔订单既满足了包邮的设置规则，又满足了购物车规则，那么设置规则优先。

第一步：设置包邮

  1. 后台侧边栏，点击店铺(Stores) > 设置(Settings) > 配置(Configuration)

  2. 在左侧面板选择销售(Sales)> 送货方式(Shipping Methods)

  3. 展开 包邮(Free Shipping) 做如下操作:
    a. 选择启用(Enabled)为是
    b. 标题(Title)字段输入结算时显示的标题
    c. 输入一个 方式名称(Method Name)来描述该送货方式
    d. 输入最小订单金额(Minimum Order Amount)，达到该值即可包邮。

    e. 在出错提示信息(Displayed Error Message)中输入包邮方式出错时的提示信息。
    f. 设置哪些国家可用(Ship to Applicable Countries)

    name

    description

    所有允许的国家All Allowed Countries

    Allow Country设置中允许的所有国家都允许

    指定国家Specific Countries

    选择本项后，下面的(Ship to Specific Countries)列表可以用了

    g. 不可用时是否显示(Show Method if Not Applicable)可选择如下选项：

    name

    description

    Yes

    就算不可以包邮也要显示这个送货方式

    No

    只有可以包邮时才有这个选项

    h. 在排序(Sort Order)中输入数字，来决定包邮选项在送货方式列表中顺序（0表示第一个，1表示第二个，依次类推）

  4. 完成后，点击 Save Config

第二步：物流公司的设置中启用免邮

如果你打算在某些物流中使用包邮的话，请确保配置好物流公司设置中关于包邮的部分。举例来说，您的UPS其他配置都完成了以后，您可以参考以下步骤来配置包邮。

  1. 在送货方式(Shipping Methods)配置中展开UPS部分
  2. 设置Free Method 为“Ground”
  3. 要设置包邮的门槛，请设置包邮门槛(Free Shipping Amount Threshold)为“Enabled”
  4. 在包邮门槛(Free Shipping Amount Threshold)中输入至少要达到的金额。
  5. 完成后，点击Save Config

Magento 的时区和时间（原理）

发表于 2016-10-12 分类于 其他

Magento 的时间存储使用的是 MySQL 的 TIMESTAMP 。TIMESTAMP 在数据库内部是以 UTC 时间存储的，在取回显示时却是依据当前时区进行转化的。

比如当前时区是UTC，存入UTC时间00：00，你看到的是00：00，但当你将时区改成UTC+8以后，你看到的时间将会是08：00，存储的值一直没有变，但是你看到的不一样了。关于这个请参考MySQL 文档： The DATE, DATETIME, and TIMESTAMP Types

重点摘要：

Magento 的系统使用的都是 UTC 时间，两个地方有体现：

第一处是 PHP 的时区设置，在 Magento 2 的入口文件 app/bootstrap.php 中有设置date_default_timezone_set('UTC');
了解更多可参考：PHP 的时间和时区。

Magento 是怎么发邮件的

发表于 2016-10-11 分类于 其他

很多用 Magento 的人可能并不是技术人员，他们也不关心 Magento 是怎么发邮件的，只要能用就好了。不过知道这些会有助于发生问题时更快定位问题所在。

Magento 发送邮件的功能是建立在 Zend Framework 的 Zend_Mail 类基础上。负责完成该工作的是 Zend_Mail_Transport_Sendmail.php ，这个类又是建立在 PHP 的 Mail 基础上的。PHP 的 Mail 使用的是服务器提供的 sendmail 脚本来发送邮件。

如果你不知道 Magento 默认是使用服务器的 sendmail 来发邮件的，那么在后台弄好邮件相关设置后，就会莫名其妙地发现 Magento 还是不能发送邮件。

如何搭建一个简单的邮件服务器（邮件传输代理）呢？可以使用 postfix、sendmail 等等。这种做法的直接好处是，可以使用许多个邮箱地址，完成不同的工作。比如：一般的邮件用 owner@yourdomain.com ，销售相关的用 sales@yourdomain.com。下面的方式就不能使用多个邮箱地址了。

下面要说到的这种方式是使用 SMTP 插件。这种插件的原理是使用新的类代替 Zend_Mail_Transport_Sendmail.php 完成工作（实际上是\Magento\Framework\Mail\Transport 负责的），这个新的类可以设置 SMTP 。这种方式相当于你把账户名，密码告诉他，他直接用外部的邮件服务器发邮件。

这种方式只能配置一个账户，所以后台邮件地址得设置地和他一样。

以上是博主的个人理解，有错误还请各位纠正，谢谢！

SMTP 插件的实现可参考：https://webkul.com/blog/magento-2-send-mail-using-your-smtp-detail/
